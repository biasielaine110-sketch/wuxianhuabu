Node 后端部署要点（Docker）

1) Railway（示例）
   - 新建 Project → Deploy from GitHub → 选本仓库。
   - Service → Settings → Root Directory 设为：backend
   - Settings → 生成域名（Generate Domain），记下 HTTPS 根地址，例如 https://xxx.up.railway.app
   - Variables：把本地 backend/.env.local 里同类变量全部拷贝进去（GOOGLE_CLOUD_PROJECT、GOOGLE_CLOUD_LOCATION、PROXY_HEADER 等）。
   - Railway 会注入 PORT；本仓库 server.js 已优先读取 process.env.PORT。
   - 构建使用 backend/Dockerfile，健康检查路径：GET /health

2) 本地仍只监听本机时
   - 在 backend/.env.local 增加：API_BACKEND_HOST=127.0.0.1

3) 与 Vercel 前端联调
   - 在 Vercel 前端环境变量设置：VITE_BACKEND_ORIGIN=https://你的Railway域名（无末尾斜杠）
   - 同时设置 VITE_PROXY_HEADER，取值与本后端 Variables 里的 PROXY_HEADER 完全一致（否则 /api-proxy 返回 403）
   - 保存后重新 Deploy 前端。

4) Vercel 部署本仓库前端（根目录已有 vercel.json）
   - New Project → 导入本 Git 仓库；Root Directory 留空（仓库根）。
   - Framework Preset 选「Other」或让 Vercel 读取 vercel.json 即可。
   - Output：frontend/dist；安装/构建命令勿在面板里覆盖，除非你有意修改。
   - Environment Variables（Production / Preview 按需）：
       VITE_BACKEND_ORIGIN = 上述后端 HTTPS 根地址（无尾斜杠）
       VITE_PROXY_HEADER = 与后端 PROXY_HEADER 相同字符串
   - 首次连接：本地在项目根执行 vercel link，或将仓库通过 Git 集成授权给 Vercel。
   - 部署后浏览器 Network 确认 Vertex 相关请求指向你的后端主机；WebSocket 走 wss://后端/ws-proxy。
